## 二、核心业务流程

### 2.1 物资申请完整链路

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  中心发起申请 │ → │  L1审批      │ → │  采购/发放   │ → │  物流在途    │ → │  确认收货    │
│  (L3)        │    │  (L1)        │    │  (L1)        │    │              │    │  (L3)        │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
                          ↓
                    ┌──────────────┐
                    │  拒绝申请    │
                    │  (通知中心)  │
                    └──────────────┘
```

### 2.2 物资发放决策逻辑

```
申请审批通过后：
    │
    ├── 检查库存
    │       │
    │       ├── 库存充足 ────→ 直接发放 ────→ 生成出库单
    │       │                         │
    │       │                         ↓
    │       │                   通知中心收货
    │       │
    │       └── 库存不足 ────→ 生成采购需求
    │                               │
    │                               ↓
    │                         采购到货 ────→ 入库 ────→ 发放
    │                                                       │
    │                                                       ↓
    └──────────────────────────────────────────────── 通知中心收货
```

### 2.3 临床物资vs日常物资审批差异

| 维度 | 临床研究物资 (clinical_supply) | 日常办公物资 (office_supply) |
|------|-------------------------------|------------------------------|
| 审批层级 | 需审核试验相关性、适用性 | 简化审批，数量限制 |
| 温控要求 | 冷链物资需温度记录仪追踪 | 无 |
| 批号追溯 | 试验用药需批号管理 | 无 |
| GCP合规 | 知情同意书等需合规审核 | 无 |
| 审批时效 | 3个工作日 | 1个工作日 |
| 紧急申请 | 支持加急（24h响应） | 不支持 |

## 三、两大物资分类详细定义

### 3.1 临床研究物资 (clinical_supply)

**定义：** 与临床试验直接相关，受GCP法规约束，需要批号追踪、温度记录的专业物资。

**二级分类及字段扩展：**

| 分类编码 | 分类名称 | 特殊字段 | 管理要求 |
|---------|---------|---------|---------|
| IP | 试验用药 (Investigational Product) | 批号、有效期、规格、存储条件(II级温控)、用药方案 | 最高管控 |
| REAGENT | 试剂盒/实验室耗材 | 批号、效期、存储条件 | 需验收记录 |
| CRF | CRF表/问卷/患者日记卡 | 版本号(ICF版本)、印刷批次 | 编号管理 |
| ICF | 知情同意书 | 版本号、IRB备案号 | 唯一性追溯 |
| DEVICE | 温度记录仪/冷链设备 | 设备编号、校准日期 | 定期校准 |
| EMERGENCY | 急救设备 | 设备编号、检查日期 | 定期检查 |

**物资编码规则：** SUPPLY-{TYPE}-{YYYY}-{NNNNNN}
- SUPPLY-IP-2024-000001 (试验用药)
- SUPPLY-CRF-2024-000001 (CRF表)
- SUPPLY-ICF-2024-000001 (知情同意书)

### 3.2 日常办公物资 (office_supply)

**定义：** 日常运营所需，走简化审批流程的通用物资。

**二级分类：**

| 分类编码 | 分类名称 | 示例物资 |
|---------|---------|---------|
| STATIONERY | 办公文具 | 笔、纸、文件夹、订书机 |
| PRINT | 打印耗材 | 硒鼓、墨盒、打印纸 |
| IT | IT设备/配件 | 键盘、鼠标、U盘 |
| DAILY | 生活用品 | 纸巾、饮用水、清洁用品 |

```
┌─────────────────┐      ┌─────────────────┐      ┌─────────────────┐
│ supply_category │ ──── │  supply_item    │ ──── │  supply_stock   │
│ 物资分类        │ 1:N  │  物资主数据     │ 1:N  │  库存           │
└─────────────────┘      └─────────────────┘      └─────────────────┘
                                │                        │
                                │                        │
                                ↓                        ↓
┌─────────────────┐      ┌─────────────────┐      ┌─────────────────┐
│ supply_supplier │      │ supply_request  │ ←── │ supply_request  │
│ 供应商          │      │  物资申请       │      │  _item          │
└─────────────────┘      └─────────────────┘      │  申请明细       │
                                │                └─────────────────┘
                                │
                                ↓
┌─────────────────┐      ┌─────────────────┐      ┌─────────────────┐
│ supply_purchase │ ←── │ supply_purchase │      │ supply_delivery │
│ _order          │      │  _item          │      │  发放/出库单   │
│ 采购订单        │      │  采购明细       │      └─────────────────┘
└─────────────────┘      └─────────────────┘              │
                                │                          │
                                ↓                          ↓
                        ┌─────────────────┐      ┌─────────────────┐
                        │ supply_receive  │      │ supply_delivery │
                        │  收货确认       │      │  _item          │
                        └─────────────────┘      │  发放明细       │
                                                 └─────────────────┘
```

### 4.2 与其他域的关联

**关联项目 (project.db)：**
- supply_request.project_id → project.id
- 物资申请必须关联项目
- 预算消耗同步到 budget.db

**关联中心 (project.db → site)：**
- supply_request.center_id → site.id
- 每个申请归属于特定中心
- L2项目视图聚合所有中心的申请

**关联合同 (contract.db)：**
- supply_purchase_order.contract_id → contract.id（可选）
- 大宗采购走合同流程

## 六、库存管理机制

### 6.1 安全库存策略

```javascript
// 库存预警规则
{
  "safety_stock_level": 100,      // 安全库存量
  "reorder_point": 50,             // 再订货点
  "reorder_quantity": 200,        // 每次采购量
  "lead_time_days": 7,             // 采购提前期
  "max_stock_level": 500          // 最大库存
}

// 预警级别
当库存 <= safety_stock_level → 黄色预警
当库存 <= reorder_point → 橙色预警（需立即采购）
当库存 <= 10% → 红色预警（紧急）
当库存 = 0 → 缺货（自动生成采购需求）
```

### 6.2 库存操作类型

| 操作类型 | 说明 | 库存变化 |
|---------|------|---------|
| 入库 (IN) | 采购到货、退回入库 | + |
| 出库 (OUT) | 发放给中心、报损 | - |
| 盘点 (CHECK) | 实际盘点调整 | +/- |
| 调拨 (TRANSFER) | 仓库间调拨 | +/- |
| 退回 (RETURN) | 中心退回 | + |

### 6.3 临床物资特殊管理

**试验用药(IP)管理：**
- 批号管理：出库时记录批号
- 效期管理：近效期(30天)预警
- 温控记录：冷链物资记录运输温度
- 用药追溯：可追溯到具体患者

## 七、申请流程状态机

### 7.1 申请状态流转

```
PENDING(待审批) → APPROVED(已审批) → PROCESSING(处理中) → SHIPPED(已发货) → RECEIVED(已收货)
      ↓               ↓                   ↓
  REJECTED(拒绝)  CANCELLED(取消)     PARTIAL(部分发货)
```

### 7.2 采购订单状态流转

```
DRAFT(草稿) → SUBMITTED(已提交) → CONFIRMED(供应商确认) → SHIPPED(已发货) → RECEIVED(已入库)
      ↓             ↓                    ↓
  CANCELLED    REJECTED(拒绝)        CANCELLED
```

### 7.3 发放单状态流转

```
PENDING(待发货) → PROCESSING(打包中) → SHIPPED(已发货) → IN_TRANSIT(运输中) → DELIVERED(已送达)
```
