# BD Module 前端页面设计 (对象线: BD / 商务拓展)

> 商务拓展域完整设计，含两入口页面、公共组件规范
> **BD项目是独立对象线，不是申办方子对象**

---

## 一、页面层级对照

| 层级 | 菜单位置 | JS文件 | 视角 | 操作对象 |
|------|---------|--------|------|---------|
| L1-BD综合管理 | 综合管理>商务拓展>BD项目管理 | bd-project-list.js | 全平台 | BD项目+拜访记录 |
| L2-BD项目详情 | BD项目>详情 | bd-project-detail.js | 单项目 | BD项目详情+跟进记录 |
| L3-拜访记录管理 | BD项目>详情>拜访记录 | visit-record-list.js | 单项目 | 拜访记录CRUD |

**配色方案：** 与已有原型统一
- Primary: #0891B2 (青色)
- Accent: #059669 (绿色)
- Background: #ECFEFF (浅蓝)
- Foreground: #164E63 (深青)

### 2.2 BD项目列表页

**筛选条件：**
- 阶段: 全部 / 初步接触 / 需求确认 / 方案报价 / 商务谈判 / 已签约 / 成功 / 失败
- 类型: 全部 / 新客户开发 / 老客户扩展
- 负责BD: 全部 / [用户选择器]
- 来源: 全部 / 展会获客 / 客户转介绍 / 主动开发 / 其他
- 状态: 全部 / 进行中 / 已归档

**列表字段：**
| 字段 | 说明 | 样式 |
|------|------|------|
| bd_project_name | BD项目名称 | 蓝色链接，点击进入详情 |
| sponsor_name | 关联申办方名称 | 灰色文字 |
| bd_stage | 当前阶段 | 阶段Tag |
| bd_type | 项目类型 | NEW_CLIENT=新客户/EXISTING_EXPANSION=老客户扩展 |
| estimated_value | 预估价值 | 格式：¥1,500万 |
| probability | 成单概率 | 百分比+进度条 |
| assigned_bd_name | 负责BD | 用户名 |
| visit_count | 拜访次数 | 数字 |
| actions | 操作 | 查看/编辑/删除/推进阶段 |

**阶段Tag颜色：**
- INITIAL: #94A3B8 (灰)
- PROSPECTING: #60A5FA (蓝)
- QUALIFICATION: #34D399 (绿)
- QUOTATION: #FBBF24 (黄)
- NEGOTIATION: #F97316 (橙)
- SIGNED: #8B5CF6 (紫)
- WON: #059669 (深绿)
- LOST: #DC2626 (红)

## 四、公共组件规范

### 4.1 BD项目选择器 (BDProjectPicker)

**使用场景：** 新建拜访记录时选择关联的BD项目

**交互：**
- 输入框，搜索BD项目名称
- 下拉显示：bd_project_name / sponsor_name / bd_stage
- 支持单选
- 可选"不关联"

**props：**
```javascript
{
  placeholder: '搜索BD项目名称',
  sponsorId: null,  // 可选，限定申办方
  onChange: (selected) => {},
  allowClear: true
}
```

### 4.2 BD项目阶段标签 (BDStageTag)

**使用场景：** BD项目列表、详情页

**交互：**
- 显示当前阶段
- 不同阶段显示不同颜色

**props：**
```javascript
{
  stage: 'NEGOTIATION',
  size: 'normal', // small/normal/large
  onClick: () => {} // 可选，点击可打开推进阶段弹窗
}
```

### 4.3 拜访类型标签 (VisitTypeTag)

**使用场景：** 拜访记录列表、详情页

**交互：**
- 显示拜访类型
- 不同类型显示不同颜色

**props：**
```javascript
{
  type: 'NEGOTIATION',
  size: 'normal'
}
```

### 4.4 拜访人员选择器 (VisitorPicker)

**使用场景：** 新建/编辑拜访记录

**交互：**
- 用户多选器
- 显示已选用户头像和姓名
- 支持搜索

**props：**
```javascript
{
  value: ['user-uuid-1', 'user-uuid-2'],
  onChange: (userIds) => {},
  maxCount: 10
}
```

### 4.5 联系人快速选择 (ContactQuickSelect)

**使用场景：** 新建/编辑拜访记录，选择客户对接人

**交互：**

## 六、BD项目阶段流转

### 6.1 阶段定义

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 初步接触 │───→│ 需求确认 │───→│ 方案报价 │───→│ 商务谈判 │───→│ 已签约   │───→│ 成功关闭 │
│ INITIAL  │    │PROSPECTING│    │QUALIFICA │    │NEGOTIATION│    │ SIGNED   │    │ WON      │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
     │              │               │               │                              
     │               │               │               │                              
     └───────────────┴───────────────┴───────────────┴───────────────┐              
                                                                        │              
                                                                        ▼              
                                                                ┌──────────┐        
                                                                │ 失败关闭 │        
                                                                │ LOST     │        
                                                                └──────────┘        
```

### 6.2 阶段说明

| 阶段 | 英文值 | 说明 | 可推进到 |
|------|--------|------|---------|
| 初步接触 | INITIAL | 初次接触客户，了解基本信息 | PROSPECTING, LOST |
| 需求确认 | PROSPECTING | 确认客户需求，评估合作意向 | QUALIFICATION, LOST |
| 方案报价 | QUALIFICATION | 提供技术方案和报价 | QUOTATION, LOST |
| 商务谈判 | QUOTATION | 商务条款谈判 | NEGOTIATION, LOST |
| 商务谈判 | NEGOTIATION | 最终商务细节确认 | SIGNED, LOST |
| 已签约 | SIGNED | 已转化为正式项目，等待启动 | WON |
| 成功 | WON | 项目正式启动 | - |
| 失败 | LOST | 合作未达成 | - |

### 6.3 阶段推进规则

1. **顺序推进**：必须按顺序推进，不能跳跃
2. **失败回退**：任何阶段可选择"失败关闭"，需填写输单原因
3. **签约转化**：NEGOTIATION阶段可点击"转化为正式项目"，自动变为SIGNED
4. **成功标记**：正式项目启动后，BD项目标记为WON
