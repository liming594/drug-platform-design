# 用印管理 Module 前端页面设计 (对象线16号: SEAL)

> 用印管理域完整设计，含三入口页面、公共组件规范

---

## 一、三界面对照

| 层级 | 菜单位置 | JS文件 | 视角 | 操作对象 |
|------|---------|--------|------|---------|
| L1-用印主档 | 综合管理>用印管理 | seal-master.js | 全平台 | 用印申请+审批+记录 |
| L2-项目用印 | 项目中心>某项目>用印管理 | project-seal.js | 单项目 | 项目用印申请+审批+记录 |
| L3-中心用印 | 中心驾驶舱(嵌入) | project-site-seal.js | 单中心 | 中心用印概览(只读) |

### 2.2 统计概览

**统计卡片（5个）：**
| 指标 | 说明 |
|------|------|
| 本月申请 | 本月提交的用印申请数量 |
| 待审批 | 当前审批节点等待中的申请 |
| 已完成用印 | 已完成盖章的申请 |
| 本月用印量 | 本月实际盖章次数(统计盖章份数) |
| 逾期未还 | 超过规定时限未归还的文件数 |

**数据来源：** GET /api/seal/stats → 聚合 seal_request + seal_usage_log

### 2.3 列表

**渲染逻辑：**
1. 页面加载 → GET /api/seal/requests → 渲染表格
2. 列：request_no / seal_type_display / applicant_name / usage_purpose / created_at / approval_status_display / current_node / actions
3. 支持分页：page/size，默认 size=20
4. 支持排序：sort=created_at&order=desc

**审批状态Tag：**
- PENDING=蓝 / APPROVING=蓝 / APPROVED=绿 / REJECTED=红 / WITHDRAWN=灰 / CANCELLED=灰

**编号规则：** YY-YYYYMM-NNNNN（如 YY-202406-00001）

### 2.4 新增用印申请弹窗

**🔴 核心设计：用印分类（项目类 / 非项目类）**

用印申请分为两大类，决定后续审批流程和数据归属：

| 分类 | 关联项目 | 审批流程 | L2项目中心可见 | 典型场景 |
|------|---------|---------|--------------|---------|
| **项目类** | ✅ 必选项目 | 项目经理 → 部门经理 → 法务 → 总经理(>50万) | ✅ 可见 | 临床合同盖章、项目协议签署 |
| **非项目类** | ❌ 不关联项目 | 部门经理 → 法务 → 总经理(>50万) | ❌ 不可见 | 公司资质文件、内部证明、公证材料 |

选择"项目类"后，项目选择器变为必填，项目自动带出到L2项目中心；选择"非项目类"后，走部门审批流程，不进项目用印列表。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| request_category | 单选 | ✅ | 项目类(PROJECT)/非项目类(NON_PROJECT) |
| project_id | 下拉选择 | ✅(项目类时) | 选择关联项目，非项目类时隐藏 |
| seal_type | 下拉选择 | ✅ | 公章/合同专用章/财务专用章/法人章 |
| usage_purpose | 下拉选择 | ✅ | 合同盖章/协议签署/授权委托书/资质文件/证明文件/公证材料/其他 |
| copy_count | 数字输入 | ✅ | 申请份数(1-100) |
| document_name | 文本输入 | ✅ | 文件名称 |
| document_count | 数字输入 | | 文档份数(默认1) |
| remark | 文本域 | | 备注说明 |
| attachment | 文件上传 | | 支持PDF/Word/图片 |

**提交逻辑：**
1. POST /api/seal/requests
2. 后端自动生成 request_no
3. **审批流程按类别创建：**
   - 项目类：项目经理 → 部门经理 → 法务 → 总经理(>50万合同)
   - 非项目类：部门经理 → 法务 → 总经理(>50万合同)
4. 项目类申请自动关联 project_id，L2项目中心可通过 project_id 筛选查看
5. 返回完整对象，前端刷新列表

### 3.4 审批流程（项目维度）

**项目用印审批流：**
- 小额(≤10万)：部门经理 → 完成用印

**特殊规则：**
- 合同盖章必须法务审批
- 授权委托书必须总经理审批

### 5.1 印章选择器 (SealPicker)

### 5.2 审批状态组件 (ApprovalStatus)

**使用场景：** 申请列表、详情页

**状态显示：**
| 状态 | 颜色 | 图标 |
|------|------|------|
| PENDING | 蓝 | ○ |
| APPROVING | 蓝 | ◐ |
| APPROVED | 绿 | ✓ |
| REJECTED | 红 | ✗ |
| WITHDRAWN | 灰 | ↩ |

**props：**
```javascript
{
  status: 'APPROVING',
  currentStep: 2,  // 当前节点索引
  totalSteps: 3    // 总节点数
}
```

### 5.3 审批流程图 (ApprovalFlow)

**使用场景：** 详情页审批流程展示

**渲染：**
- 横向步骤条，显示各审批节点
- 当前节点高亮
- 已完成节点显示✓

  steps: [
