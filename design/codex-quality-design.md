### 2.2 仪表盘卡片

**渲染逻辑：**
1. 页面加载 → GET /api/quality/dashboard → 渲染概览卡片
2. 卡片数据：待处理(OPEN) / 进行中(IN_PROGRESS) / 待审批(PENDING_APPROVAL) / 已闭环(CLOSED) / 超期(overdue) / 闭环率(closed_count/total_count×100%)

**卡片颜色：**
- 待处理：蓝
- 进行中：橙
- 待审批：黄
- 已闭环：绿
- 超期：红

**质量指标趋势：**
- 偏差率：本月偏差数/本月总入组数
- CAPA闭环率：本月已闭环CAPA/本月应闭环CAPA
- 调用 GET /api/quality/indicators?period=month 获取趋势数据

### 2.3 发现列表

**渲染逻辑：**
1. 页面加载 → GET /api/quality/findings?page=&size=&type=&severity=&status=&project_id=&keyword=
2. 列：finding_no / project_name / finding_type / severity / site_name / found_date / capa_status / 操作
3. 状态Tag颜色：
   - OPEN=蓝(待处理)
   - IN_PROGRESS=橙(CAPA进行中)
   - PENDING_APPROVAL=黄(待审批)
   - VERIFIED=绿(已验证闭环)
   - CLOSED=深绿

**严重程度Tag：**
- CRITICAL=红(关键偏差)
- MAJOR=橙(主要不符)
- MINOR=黄(轻微偏差)
- OBSERVATION=灰(观察项)

**超期标记：** 发现日期+计划闭环日期超过设定阈值，行标红+告警图标

### 3.2 项目级特有组件

**中心质量排名：**
- 横向条形图，按本项目各中心发现数量排序
- 数据：GET /api/quality/findings?project_id=xxx&group=site
- 显示：site_name / finding_count / 排名

**质量指标趋势：**
- 折线图+柱状图组合
- 指标：本月偏差率 / CAPA闭环率 / 及时整改率
- 数据：GET /api/quality/indicators?project_id=xxx&period=month

**与L1区别：**
- 默认筛选 project_id = 当前项目
- 中心下拉筛选器默认展示本项目所有中心
- 指标数据限定本项目范围

## 七、交互逻辑说明

### 7.1 发现状态流转

```
OPEN → IN_PROGRESS (当有CAPA被创建)
    ↓
PENDING_APPROVAL (所有CAPA验证通过)
    ↓
VERIFIED (验证人确认)
    ↓
CLOSED (发现正式关闭)
```

**状态变更时机：**
- OPEN → IN_PROGRESS：POST capa时自动触发
- IN_PROGRESS → PENDING_APPROVAL：所有capa.status=VERIFIED时自动触发
- PENDING_APPROVAL → VERIFIED：验证人手动确认
- VERIFIED → CLOSED：详情页点击"关闭"

### 7.2 CAPA状态流转

```
PENDING → IN_PROGRESS (开始执行)
    ↓
PENDING_VERIFICATION (提交完成)
    ↓
VERIFIED (验证通过)
    ↓
CLOSED (措施闭环)
```

**反驳流程：** VERIFIED → IN_PROGRESS（验证人打回，需填写原因）

### 7.3 超期告警逻辑

**发现超期：** found_date + 30天 < today AND finding_status ≠ VERIFIED/CLOSED
**CAPA超期：** due_date < today AND capa_status ≠ VERIFIED/CLOSED

**告警方式：**
- 行内标红
- 仪表盘超期数字+1
- 站内通知发送给：发现负责人 + CAPA负责人 + PM

### 7.4 闭环率计算

```
闭环率 = 已闭环CAPA数 / 应闭环CAPA数 × 100%
应闭环CAPA = 已创建CAPA - 超期未闭环(分子不含超期)
```

**时间范围：** 可选 本月/本季度/本年，默认本月
