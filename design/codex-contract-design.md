# Contract Module 前端页面设计 (对象线17号: Contract / 合同管理)

> 合同管理域完整设计，含三入口页面、公共组件规范
> **独立库零耦合：contract.db，不与其他库共享表，通过ID引用关联**
> 数据库名：contract.db

---

## 一、三界面对照

| 层级 | 菜单位置 | JS文件 | 视角 | 操作对象 |
|------|---------|--------|------|---------|
| L1-合同主档 | 综合管理>合同管理 | contract-L1-master.html | 全平台 | 全平台合同+收款+开票(可CRUD) |
| L2-项目合同 | 项目中心>某项目>合同管理 | contract-L2-project.html | 单项目 | 项目关联合同+收款+开票 |
| L3-合作方合同 | 各合作主体详情中 | contract-L3-partner.html | 单合作方 | 4个Tab: 申办方/SMO/中心/合作方(只读) |

> **🔴 重要调整：L3为统一4入口页面**
> - L3合并为1个HTML文件，通过Tab切换展示4类合作方合同
> - 申办方合同 / SMO合同 / 中心合同 / 合作方合同 4个Tab
> - 所有L3入口均只读，不可新建/编辑/删除

### 2.2 仪表盘卡片

**渲染逻辑：**
1. 页面加载 → GET /api/contract/dashboard → 渲染概览卡片
2. 卡片数据：合同总数(execute+pending) / 执行中(ACTIVE) / 待签署(PENDING_SIGN) / 本月到期 / 应收未收 / 已收总额

**卡片颜色：**
- 合同总数：蓝
- 执行中：绿
- 待签署：黄
- 本月到期：橙
- 应收未收：红
- 已收总额：深蓝

**收款进度条：**
- 总应收 = SUM(contract.contract_amount WHERE contract_status NOT IN (TERMINATED))
- 已收 = SUM(contract_collection.collection_amount)
- 收款率 = 已收 / 总应收 × 100%

### 2.3 合同列表

**渲染逻辑：**
1. 页面加载 → GET /api/contract/contracts?page=&size=&party_type=&contract_type=&status=&project_id=&keyword=
2. 列：contract_no / contract_name / party_name / contract_type / contract_amount / contract_status / sign_date / 操作
3. 状态Tag颜色：
   - DRAFT=灰(草稿)
   - PENDING_APPROVAL=黄(待审批)
   - PENDING_SIGN=橙(待签署)
   - ACTIVE=绿(执行中)
   - CHANGING=蓝(变更中)
   - EXPIRED=红(已到期)
   - TERMINATED=深灰(已终止)

**合同类型Tag：**
- MAIN=蓝(主合同)
- SUPPLEMENT=绿(补充协议)
- NDA=紫(NDA)
- FRAMEWORK=橙(框架协议)

## 六、财务同步机制

### 6.1 设计原则：Single Source of Truth

**核心定位：**
- 合同数据唯一源头在 `contract.db`，合同管理模块是唯一可CRUD的入口
- 财务模块（`finance.db` / `budget.db`）中的合同数据是**引用视图**，通过API同步读取
- 财务同事在财务入口看到的是：合同概要+收款+付款+发票放一起，方便操作
- 财务入口对合同本身是**只读**的（合同编辑只能从合同管理进入）

### 6.2 数据同步规则

**事件驱动同步：**
```
合同创建 → 事件触发 → 同步合同概要到 finance.db
合同变更 → 事件触发 → 同步变更字段到 finance.db
收款确认 → 事件触发 → 更新 finance.db 关联记录
```

**同步触发时机：**
| 操作 | 触发条件 | 同步内容 |
|------|---------|---------|
| 合同新建 | POST /api/contract/contracts 成功 | 同步合同基础信息 |
| 合同编辑 | PUT /api/contract/contracts/:id 成功 | 同步变更字段 |
| 合同状态变更 | PUT /api/contract/contracts/:id/status | 同步状态变更 |
| 收款确认 | POST /api/contract/collections 成功 | 更新已收金额 |
| 发票开具 | POST /api/contract/invoices 状态变更为ISSUED | 更新已开票金额 |
| 合同终止 | 状态变更为 TERMINATED | 同步终止状态 |

### 6.3 同步字段定义

**同步到 finance.db 的字段映射：**

| 源字段 (contract.db) | 目标字段 (finance.db) | 说明 |
|----------------------|----------------------|------|
| contract_id | ref_id / contract_id | 合同唯一标识 |
| contract_no | contract_no | 合同编号 |
| contract_name | contract_name | 合同名称 |
| party_name | counterparty_name | 合作方名称 |
| contract_amount | total_amount | 合同总金额 |
| collected_amount | paid_amount | 已收款金额 |
| invoiced_amount | invoiced_amount | 已开票金额 |
| contract_status | status | 合同状态 |
| sign_date | sign_date | 签署日期 |
| expiry_date | expiry_date | 到期日期 |

**finance.db 需新增/扩展的表：**

```sql
-- finance_contract_view：财务模块的合同引用视图（不存储，只读同步）
CREATE TABLE IF NOT EXISTS finance_contract_view (
    view_id          TEXT PRIMARY KEY,
    contract_id      TEXT NOT NULL,
    contract_no     TEXT NOT NULL,
    contract_name   TEXT NOT NULL,
    party_name      TEXT,
    total_amount    REAL DEFAULT 0,
    paid_amount     REAL DEFAULT 0,
    invoiced_amount REAL DEFAULT 0,
    pending_amount  REAL DEFAULT 0,
    status          TEXT,

    expiry_date     TEXT,

```

### 6.4 同步方向：单向 Contract → Finance

```
┌─────────────────┐         同步          ┌─────────────────┐
│   contract.db  │ ───────────────────→  │   finance.db    │
│   (合同主库)    │   事件驱动单向同步    │   (财务视图)    │
└────────┬────────┘                       └────────┬────────┘
         │                                         │
         │ 唯一CRUD入口                             │ 只读引用
         │                                         ↓
         │                               ┌───────────────────┐
         └─────────────────────────────→ │  财务入口页面      │
                                         │  (合同只读展示)    │
                                         └───────────────────┘
```

**同步是单向的：**
- 财务模块**不直接写** contract.db
- 财务模块的操作（如付款确认）→ 调用 contract API → contract 触发同步事件 → 更新 finance.db

### 6.5 冲突处理机制

**冲突定义：** finance.db 中的合同数据与 contract.db 不一致

**冲突检测：**
```javascript
// 对比 contract.db 最新数据与 finance.db 缓存
function detectConflict(contractId) {
    const master = contractDb.get(contractId);      // contract.db 主数据
    const view = financeDb.getByContractId(contractId);  // finance.db 缓存
    
    if (master.updated_at > view.last_sync_time) {
        return { hasConflict: true, master, view };
    }
    return { hasConflict: false };
}
```

**冲突处理策略：**

| 场景 | 处理方式 | 说明 |
|------|---------|------|
| 定期校对 | 每日凌晨比对 contract.db 与 finance.db | 发现差异自动校正 |
| 实时检测 | 前端显示同步状态指示器 | 冲突时标红提示 |
| 强制覆盖 | 以 contract.db 为准 | finance.db 定期全量同步 |
| 人工确认 | 冲突时暂停同步，等待人工确认 | 仅在极端情况下触发 |

**定期校对任务（每日凌晨执行）：**
```javascript
// 同步任务
async function dailySyncTask() {
    const allContracts = await contractDb.getAll();
    for (const contract of allContracts) {
        const existing = await financeDb.getByContractId(contract.contract_id);
        if (!existing) {
            // 新增：插入finance视图
            await financeDb.insert(mapToFinanceView(contract));
        } else if (contract.updated_at > existing.last_sync_time) {
            // 更新：同步变更
            await financeDb.update(contract.contract_id, mapToFinanceView(contract));
        }
    }
    // 清理：删除finance中contract已不存在的记录
    await cleanupOrphanedRecords();
}
```

### 6.6 财务视角：合同在财务模块的展示

**财务模块（finance.db）中的合同展示规则：**

**L1 财务总览（综合管理 > 财务 > 合同总览）：**
- 展示合同总览卡片：合同总数/执行中/待收款/已收款
- 展示应收未收排行榜
- 点击合同行 → 跳转合同详情（只读弹窗）
- 🔴 **新增：财务同步状态指示器**

**L2 项目财务（项目中心 > 某项目 > 财务）：**
- 项目合同概览：合同总金额/已收/未收/开票金额
- 合同作为收款来源展示
- 收款记录关联合同

**财务同步状态指示器：**
| 状态 | 图标 | 颜色 | 说明 |
|------|------|------|------|
| SYNCED | ✅ | 绿 | 已同步，与contract.db一致 |

## 七、交互逻辑说明

### 7.1 合同状态流转

```
DRAFT(草稿) → PENDING_APPROVAL(待审批) → PENDING_SIGN(待签署) → ACTIVE(执行中)
                    ↓                                                    ↓
              TERMINATED(终止)                                    CHANGING(变更中)
                                                                      ↓
                                                              ACTIVE(执行中)
                                                                      ↓
                                                              EXPIRED(到期)
```

**状态变更时机：**
- DRAFT → PENDING_APPROVAL：提交审批
- PENDING_APPROVAL → PENDING_SIGN：审批通过
- PENDING_SIGN → ACTIVE：双方签署完成
- ACTIVE → CHANGING：发起变更
- CHANGING → ACTIVE：变更审批通过
- ACTIVE → EXPIRED：到期日期到达
- PENDING_APPROVAL/PENDING_SIGN/ACTIVE → TERMINATED：提前终止

### 7.2 收款状态计算

```
计划收款日期 < today AND 未收款 = OVERDUE(逾期)
实际收款金额 > 0 AND < 计划金额 = PARTIAL(部分收款)
实际收款金额 >= 计划金额 = RECEIVED(已收)
```

**收款确认后：**
- 自动更新 contract.collected_amount
- 自动计算 contract.pending_amount
- 发送通知给项目经理

### 7.3 发票状态流转

```
PENDING(待开票) → APPROVED(已审批) → ISSUED(已开具)
                                          ↓
                                    CANCELLED(作废)
```

**作废发票：**
- 作废后生成新发票（关联原合同）
- 原发票标记为 CANCELLED
- 不冲减合同已开票金额（需重新开票）

### 7.4 合同金额冗余同步

**冗余字段（定期同步）：**
| 字段 | 来源 | 说明 |
|------|------|------|
| contract.collected_amount | SUM(collection) | 已收款 |
| contract.invoiced_amount | SUM(invoice) | 已开票 |
| contract.pending_amount | contract_amount - collected_amount | 应收未收 |
| sponsor_master.total_contract_amount | SUM(contract) | 申办方合同总额 |
| project_sponsor.total_contract_amount | SUM(project contract) | 项目合同总额 |
