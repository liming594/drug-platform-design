### 2.2 左侧-公司树

**渲染逻辑：**
1. 页面加载 → GET /api/hr/companies → 渲染公司列表
2. 点击公司 → GET /api/hr/departments?company_id=xxx → 展开部门树
3. 部门树支持多级嵌套（children递归渲染）
4. 点击部门 → 右侧更新部门详情+人员列表
5. 当前选中项高亮

**公司节点操作：**
- 点击：切换右侧内容
- 右键/操作按钮：编辑公司、新增部门（parent_id预填根级）

**部门节点操作：**
- 点击：切换右侧内容
- 右键/操作按钮：编辑部门、新增子部门（parent_id预填当前部门）

**新增公司弹窗：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| company_name | 文本输入 | ✅ | 公司全称 |
| short_name | 文本输入 | | 公司简称 |
| credit_code | 文本输入 | | 统一社会信用代码 |
| legal_person | 文本输入 | | 法人 |
| address | 文本输入 | | 注册地址 |
| contact_phone | 文本输入 | | 联系电话 |
| contact_email | 文本输入 | | 联系邮箱 |

提交 → POST /api/hr/companies → 刷新公司树

### 2.3 右上-部门列表

**渲染逻辑：**
- 选中某公司后，展示该公司的部门列表（平铺）
- 显示：department_name / member_count(JOIN统计) / manager_name(JOIN member)

**新增部门弹窗：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| department_name | 文本输入 | ✅ | 部门名称 |
| company_id | 隐藏/只读 | ✅ | 当前选中公司 |
| parent_id | 下拉选择 | | 上级部门（部门树选择器） |
| dept_type | 下拉选择 | | NORMAL/VIRTUAL，默认NORMAL |
| manager_id | 人员选择器 | | 部门负责人 |
| sort_order | 数字输入 | | 排序号 |

提交 → POST /api/hr/departments → 刷新部门列表

**删除部门：** 确认弹窗 → DELETE /api/hr/departments/:departmentId
- 有子人员时提示"该部门下有N名人员，无法删除"

| birth_date | 日期选择 | | 出生年月 |
| id_type | 下拉选择 | | ID_CARD/PASSPORT/OTHER |
| id_number | 文本输入 | | 证件号 |
| company_id | 下拉/只读 | ✅ | 默认当前选中公司 |
| department_id | 下拉/只读 | ✅ | 默认当前选中部门 |
| position_type | 下拉选择 | ✅ | CRA/CRC/PM/医学写作/CTA/DM/SA/QA/BD/其他 |
| position_level | 下拉选择 | ✅ | 初级/中级/高级/资深/专家 |

|---|---|
| 项目名 | project_name |
| 角色 | project_role |

| 状态 | assignment_status |

**变更状态弹窗：**
- 下拉选择：ACTIVE / PROBATION / RESIGNED / DISABLED
- PUT /api/hr/members/:memberId/status

### 4.3 离职详情弹窗（核心）

```
┌───────────────────────────────────────────────────┐
│  离职详情 - 陈九                                    │
├───────────────────────────────────────────────────┤
│  基本信息                                           │
│  姓名: 陈九   部门: 临床部   岗位: 高级CRA        │
│  离职日: 6/20  类型: 主动辞职   原因: 个人发展     │
│                                                    │
│  ── 分配交接 ───────────────────────────           │
│  [全选] [批量交接给同一人▼]                         │
│  ☑ PD-001 · 北京协和 · CRA  → 交接给 [选人▼]     │
│  ☑ PD-001 · 北医三院 · CRA  → 交接给 [选人▼]     │
│  ☐ PD-002 · 阜外医院 · CRA  → 交接给 [选人▼]     │
│  ☑ PD-002 · PM(40%)        → 交接给 [选人▼]     │
│  [已选3条 → 批量交接给: [选人▼] [确认]]           │
│                                                    │
│  ── 离职待办 ──────────────────────────           │
│  □ 回收系统权限                                    │
│  □ 归还办公设备                                    │
│  ☑ 结算当月工资                                    │
│  □ 档案转移                                        │
│  进度: 1/5  |  已交接分配: 0/4                     │
│                                                    │
│  [编辑]              [完成离职]                     │
└───────────────────────────────────────────────────┘
```

**分配交接区：**

1. **逐条交接：** 每条右侧[选人▼] → 确认 → PUT /api/hr/offboarding/:id/transfer/:assignmentId
   - 后端：原记录REPLACED + 新建记录给接手人
   - 交接完成后该行标记"已交接"，选人框变只读

2. **批量交接：** 勾选多条 → 底部[已选N条 → 批量交接给: [选人▼] [确认]]
   - POST /api/hr/offboarding/:id/transfer-batch
   - body: { assignment_ids: [...], new_member_id: "uuid", transfer_date: "2026-06-20" }

3. **全选：** 表头全选checkbox，全选后批量交接区激活

**离职待办区：**
- 默认项：回收系统权限 / 归还办公设备 / 结算当月工资 / 档案转移 / 交接工作文档
- 可增删改

**[完成离职]按钮：**
- 前端校验：提示"还有N条分配未交接，完成后未交接分配将被释放，确认？"
- 确认 → PUT /api/hr/offboarding/:id/complete
- 后端：offboard_status→COMPLETED，Member→RESIGNED，未交接ACTIVE→RELEASED

## 七、公共组件规范

### 7.1 人员选择器 (MemberPicker)

**使用场景：** 汇报上级、部门负责人、离职交接、项目添加人员等

**交互：**
- 输入框，搜索姓名/工号/手机号
- 下拉显示：name / employee_no / position_type / 当前负载(member_total_allocation)
- 支持单选/多选模式

**props：**
```javascript
{
  mode: 'single' | 'multi',
  placeholder: '搜索姓名/工号',
  filters: { department_id: '', position_type: '', status: 'ACTIVE' },
  onChange: (selected) => {}
}
```

### 7.2 部门树选择器 (DeptTreePicker)

**交互：** 树形结构，展开/收起，点击选中，可选显示人员数量

**props：**
```javascript
{
  companyId: '',
  showMemberCount: true,
  onSelect: (dept) => {}
}
```

### 7.3 Checklist组件 (ChecklistEditor)

**交互：**
- 渲染 [{item, done, doneDate}] 数组
- 勾选 done=true, doneDate=今天；取消 done=false, doneDate=null
- 支持增删行
- 底部进度 "2/5"

**props：**
```javascript
{
  items: [{item: '', done: false, doneDate: null}],
  editable: true,
  onChange: (items) => {}
}
```
