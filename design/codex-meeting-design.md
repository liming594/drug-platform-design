# Meeting Module 前端页面设计 (对象线15号: Meeting)

> 会议管理域完整设计，含三入口页面、公共组件规范

---

## 一、三界面对照

| 层级 | 菜单位置 | JS文件 | 视角 | 操作对象 |
|------|---------|--------|------|---------|
| L1-会议主档 | 综合管理>会议管理 | meeting-master.js | 全平台 | 会议列表+会议管理+纪要+决议 |
| L2-项目会议 | 项目中心>某项目>会议管理 | project-meeting.js | 单项目 | 项目维度会议列表 |
| L3-中心会议 | 中心驾驶舱(嵌入) | project-site-meeting.js | 单中心 | 中心维度会议信息(只读) |

### 2.2 概览卡片

**渲染逻辑：**
1. 页面加载 → GET /api/meetings/stats → 渲染卡片
2. 卡片数据：monthly_count / upcoming_count / pending_minutes / pending_resolutions

**卡片样式：**
- 数字使用大字号显示
- 卡片hover时显示环比变化
- 红色预警样式用于待跟进决议>0

### 2.3 会议列表

**筛选条件：**
| 筛选 | 类型 | 说明 |
|------|------|------|
| 会议类型 | 下拉多选 | PROJECT_INITIATION/KICKOFF/MONITORING/DATA_REVIEW/CLOSE_OUT/OTHER |
| 项目 | 下拉选择 | 从项目列表选择，跨库调用project.db |
| 日期范围 | 日期区间 | 会议开始日期 |
| 会议状态 | 下拉多选 | SCHEDULED/IN_PROGRESS/COMPLETED/CANCELLED |
| 关键词 | 文本搜索 | 搜索会议名称 |

**渲染逻辑：**
1. GET /api/meetings → 渲染列表
2. 按会议日期分组显示（今天/明天/本周/本月/更早）
3. 状态Tag颜色：SCHEDULED=蓝 / IN_PROGRESS=绿 / COMPLETED=灰 / CANCELLED=红

**列表字段：**
| 字段 | 说明 | 显示规则 |
|------|------|---------|
| meeting_time | 会议时间 | HH:mm格式，带日期分组 |
| meeting_title | 会议名称 | 加粗显示 |
| project_name | 项目名称 | 显示项目简称 |
| meeting_type | 会议类型 | 中文标签 |
| location | 会议地点 | 🏢线上 / 📍+地点名 |
| host_name | 主持人 | 显示姓名 |
| participant_count | 参会人数 | 👥+人数 |
| minutes_status | 纪要状态 | 📋已完成/待写 |
| resolution_status | 决议状态 | ⚠️X项待跟进/✅无待跟进 |

**议程项结构：**
```json
[{"topic": "汇报项目进展", "duration_minutes": 30, "responsible_id": "memberId"}, ...]
```

**提交逻辑：**
1. POST /api/meetings → 创建会议
2. POST /api/meetings/:meetingId/agenda → 创建议程
3. POST /api/meetings/:meetingId/participants → 添加参会人员
4. POST /api/meetings/:meetingId/materials → 上传材料
5. **发送通知** → 站内信/邮件通知参会人员

### 2.8 取消会议弹窗

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|

| related_type | 单选 | | 项目里程碑/伦理审批/监查计划/其他 |
| remark | 文本域 | | 备注 |

**提交后：**
1. INSERT meeting_resolution
2. **发送通知** → 通知负责人有新待办

### 2.10 决议状态变更

**完成决议弹窗：**
