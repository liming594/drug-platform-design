# 财务模块设计文档

> 模块：财务（第9号线 预算及执行 + 第10号线 费用管理）
> 数据库：budget.db（7表）+ finance.db（7表），双库独立零耦合
> 核心原则：预算审批锁定、执行自动汇总、增改全留痕、花钱走审批、预算强控
> 入口三线：综合管理(审批/监控) | 项目中心(提请/查看) | 中心驾驶舱(只读)
> 🔴 项目预算到项目级，不拆中心预算；中心预算后续需要时再补一层

---

## 一、模块定位与关系

财务模块两条线，边界清晰：**管预算的**（9号线）和**花钱的**（10号线）。

```
综合管理 > 财务
├── 预算及执行          ← 第9号线（预算审批+执行监控+费用录入+人工成本）
└── 费用管理 ★          ← 第10号线（报销/付款/借款/发票）

项目中心 > 某项目
├── 预算及执行 ★        ← 提请+查看（第9号线）
└── 费用 ★             ← 项目费用（第10号线）

中心驾驶舱 > 某中心
├── 预算及执行 ★        ← 只读（第9号线）
└── 费用 ★             ← 中心费用（第10号线）
```

**两条线的联动：**

```
第9号线 预算及执行              第10号线 费用管理
审批锁定预算+费用归集            报销/付款实际花钱
┌──────────────┐               ┌──────────────┐
│ budget       │──预算校验──→  │ expense_report│
│ budget_item  │  (超预算预警) │ payment       │
│ budget_      │←─审批通过───  │ loan         │
│ execution    │  (同步费用)   │ invoice      │
│ cost_record  │←─写费用记录── │              │
└──────────────┘               └──────────────┘
```

**核心链路：** 预算审批→锁定→花钱→审批→写费用→对比预算

## 二、第9号线 预算及执行

### 2.1 入口与职责

```
入口一：项目中心 > 某项目 > 预算及执行 ★ 提请+查看
├── 提请项目预算（起草→提交审批）
├── 查看本项目预算详情+执行
├── 人工成本（自动算）
├── 其他费用（手动录）
├── 申请变更
└── [费用管理→] 跳转第10号线本项目费用

入口二：中心驾驶舱 > 某中心 > 预算及执行 ★ 查看
├── 本中心相关费用汇总
├── 本中心人工成本
└── 只读视角

入口三：综合管理 > 财务 > 预算及执行 ★ 审批/监控
├── 项目预算列表（全局视角，审批操作）
├── 部门预算列表（提请+审批+监控）
├── 执行监控（全局）
├── 变更审批
└── [费用管理→] 跳转第10号线
```

**核心原则：谁干活谁提请，管钱的审批监控**

| 职责 | 项目中心 | 中心驾驶舱 | 综合管理 |
|------|---------|-----------|---------|
| 提请项目预算 | ✅ | ❌ | ❌ |
| 编辑预算项 | ✅（草稿/打回时） | ❌ | ❌ |
| 审批预算 | ❌ | ❌ | ✅ |
| 查看本项目执行 | ✅ | — | ✅ |
| 查看本中心费用 | — | ✅（只读） | ✅ |
| 变更申请 | ✅ | ❌ | ✅ |
| 部门预算 | ❌ | ❌ | ✅（提请+管理） |
| 人工成本/其他费用 | ✅（项目内） | ✅（中心内只读） | ✅（全局） |

**三入口数据同源：** 读写同一套budget.db表，筛选条件不同：
- 项目中心：WHERE budget_type='PROJECT' AND ref_id=:projectId
- 中心驾驶舱：WHERE cost_record.project_site_id=:siteId（只读）
- 综合管理：无筛选，全局视角

### 2.2 数据库表（budget.db，7表）

| 表名 | 用途 |
|------|------|
| budget_category | 费用类别字典（预置32类，DIRECT/INDIRECT） |
| budget | 预算主表（PROJECT/DEPARTMENT，含状态流转） |
| budget_item | 预算项明细（按类别拆分） |
| budget_approval_log | 审批日志（完整留痕） |
| budget_change | 预算变更记录（版本对比） |
| budget_execution | 执行汇总快照（按类别按日期） |
| cost_record | 费用明细记录（AUTO_LABOR/AUTO_COST/MANUAL三种来源） |

### 2.3 预算状态流转

```
DRAFT(草稿) → SUBMITTED(已提交) → APPROVING(审批中) → APPROVED(已批准/锁定)
                                     ↓                    ↓
                                  REJECTED(已打回)    CHANGING(变更中) → APPROVING → APPROVED(v2)
                                                                              ↓
                                                                        CLOSED(已关闭)
```

### 2.4 审批角色

| 角色 | 审批范围 |
|------|---------|
| SUBMITTER | 提交/撤回（项目PM或部门负责人） |
| APPROVER | 部门经理审批 |
| FINANCE | 财务审核 |
| ADMIN | 管理员（代操作/关闭/调整） |

| 预算类型 | 审批链 |
|---------|--------|
| 项目预算 | PM提交 → 部门经理 → 财务 |
| 部门预算 | 部门负责人提交 → 财务总监 → 总经理 |

### 2.5 执行数据三种来源

- **AUTO_LABOR**：工时系统自动算人工成本（timesheet.db × member.db）
- **AUTO_COST**：第10号线审批通过自动同步
- **MANUAL**：手动录入

> cost_record(逐笔) → 按category_id聚合 → budget_execution(按类汇总)

### 2.6 人工成本计算规则

- 数据源：weekly_timesheet(timesheet.db) × member_salary(member.db)
- 公式：hourly_rate = (base_salary + social_insurance_company + housing_fund_company) / 21.75 / 8
- 项目人工成本 = Σ(hourly_rate × actual_hours)，按项目×人员聚合
- 每月自动生成cost_record(source_type=AUTO_LABOR)并刷新budget_execution
- ⚠️ 个人时薪不展示，前端只看汇总数据

### 2.7 执行预警

- 执行率>80%标黄
- 执行率>100%标红
- 第10号线提交报销/付款时自动校验，超80%预警，超100%阻断需CFO特批

### 2.8 界面设计

#### 项目中心 > 某项目 > 预算及执行

5个Tab：
1. **预算明细**：本项目预算+预算项列表+操作按钮（编辑/提交/变更）
2. **执行监控**：预算vs实际对比表+执行趋势+直接/间接成本饼图
3. **人工成本**：按人员/岗位汇总，自动从工时×时薪算
4. **其他费用**：手动录入+自动同步，按来源筛选
5. **费用记录**：全部费用流水，可导出

#### 中心驾驶舱 > 某中心 > 预算及执行

- 只读：中心费用概览+中心人工成本
- 中心数据筛选：cost_record WHERE project_site_id=:siteId

#### 综合管理 > 财务 > 预算及执行

- 全局概览：总额/执行/偏差/待办
- 项目预算列表+详情（审批操作）
- 部门预算列表+详情（提请+审批）
- 执行预警面板
- 人工成本概览（全局）

## 三、第10号线 费用管理

### 3.1 入口与职责

```
入口一：综合管理 > 财务 > 费用管理 ★ 全局
├── 费用报销         员工报销差旅/培训/其他
├── 付款管理         供应商/中心/SMO付款
├── 借款管理         预借款+冲抵
└── 发票管理         收票/验真/关联

入口二：项目中心 > 某项目 > 费用 ★ 项目视角
├── 项目报销（project_id自动带入）
├── 项目付款
├── 项目借款
└── 预算执行概览（[→跳转第9号线]）

入口三：中心驾驶舱 > 某中心 > 费用 ★ 中心视角
├── 中心收款（管理费/检查费等）
├── 中心报销
└── 只读+发起视角，审批回综合管理
```

**三入口数据同源：** 所有入口读写同一套finance.db表
- 综合管理：无筛选，看全部
- 项目中心：WHERE project_id = :projectId
- 中心驾驶舱：WHERE project_site_id = :siteId / payee_type='SITE' AND payee_id=:siteId

### 3.2 数据库表（finance.db，7表）

| 表名 | 用途 |
|------|------|
| expense_report | 报销单（差旅/培训/办公/其他） |
| expense_item | 报销明细（按费用类别拆分） |
| payment | 付款单（供应商/中心/SMO/员工/其他） |
| loan | 借款单（含冲抵状态） |
| loan_repayment | 借款冲抵记录 |
| invoice | 发票（增值税专票/普票/收据，含验真） |
| finance_approval_log | 通用审批日志 |

### 3.3 审批状态流转（报销/付款/借款共用）

```
DRAFT(草稿) → SUBMITTED(已提交) → APPROVING(审批中) → APPROVED(已批准)
                                     ↓                    ↓
                                  REJECTED(已打回)    付款: PAID(已付款)
                                                     报销: PAID(已打款)
                                                     借款: PAID(已打款)
                                                              ↓
                                                        CLOSED(已关闭)
```

借款额外状态：PARTIAL_REPAID(部分冲抵) / REPAID(已清) / OVERDUE(逾期)

### 3.4 审批角色

| 角色 | 审批范围 |
|------|---------|
| SUBMITTER | 提交/撤回（申请人本人） |
| MANAGER | 部门经理审批 |
| FINANCE | 财务审核 |
| CFO | 大额审批（>¥50,000） |
| ADMIN | 管理员 |

### 3.5 金额分级审批

| 金额范围 | 审批链 |
|---------|--------|
| ≤¥5,000 | 部门经理 → 财务 |
| ¥5,001~¥50,000 | 部门经理 → 财务 → 财务总监 |
| >¥50,000 | 部门经理 → 财务 → CFO → 总经理 |

### 3.6 报销类型与费用类别映射

| 报销类型 | 可选费用类别 |
|---------|------------|
| TRAVEL 差旅 | 差旅费、住宿费、市内交通 |
| TRAINING 培训 | 培训费、会议费、差旅费 |
| OFFICE 办公 | 印刷快递费、物料耗材费、设备费 |
| OTHER 其他 | 其他直接成本、其他间接成本 |

### 3.7 与第9号线联动规则

1. **预算校验**：提交报销/付款时自动校验budget_execution，超80%预警，超100%阻断（需CFO特批）
2. **审批通过→同步费用**：自动写入budget.db的cost_record(source_type=AUTO_COST)→刷新budget_execution
3. **借款冲抵闭环**：借款→报销→冲抵，逾期自动提醒
4. **发票关联**：报销/付款可关联发票，发票状态同步更新为LINKED

### 3.8 界面设计

#### 综合管理 > 财务 > 费用管理

首页4个卡片（报销/付款/借款/发票）+ 我的待办 + 近期流水

- **费用报销**：列表+新建（含预算校验）+报销明细
- **付款管理**：列表+新建（收款方信息+关联发票+预算校验）+确认付款
- **借款管理**：列表+新建+冲抵操作+逾期提醒
- **发票管理**：列表+录入+验真+批量关联

#### 项目中心 > 某项目 > 费用

- 预算执行概览（总预算/已执行/执行率）
- 3个卡片：项目报销/项目付款/项目借款
- 近期费用流水
- 可直接新建（project_id自动带入）

#### 中心驾驶舱 > 某中心 > 费用

- 中心收款记录（payee_type=SITE）
- 中心报销（project_site_id关联）
- 只读+发起视角

## 四、通用规则

1. **谁干活谁提请**：项目预算从项目中心提请，部门预算从综合管理提请
2. **管钱的审批监控**：综合管理→财务是审批和全局监控入口
3. **预算审批后锁定**：APPROVED不可编辑，改必须走变更流程
4. **变更全留痕**：每次变更生成新版本，保留历史快照
5. **执行数据三种来源**：AUTO_LABOR + AUTO_COST + MANUAL
6. **人工成本自动算**：后端从timesheet.db×member.db读，前端不碰薪资原始数据
7. **个人时薪不展示**：前端只看汇总金额
8. **三入口数据同源**：项目中心/中心驾驶舱/综合管理读写同一套表
9. **打回必填原因**：comment必填
10. **执行预警**：>80%标黄，>100%标红
11. **发票先收后付**：付款单优先关联已验真发票，无发票可先付后补
12. **借款冲抵闭环**：借款→报销→冲抵，逾期自动提醒
13. **金额校验**：发票金额≥报销金额/付款金额，一张发票只能关联一个单据
14. **自动编号**：BX-YYYY-NNNNN（报销）、FK-YYYY-NNNNN（付款）、JK-YYYY-NNNNN（借款）、FP-YYYY-NNNNN（发票）
15. **权限控制**：个人只能看自己的报销/借款，财务/管理员看全部；人工成本仅经理层/总监/财务可见
16. **🔴 项目预算到项目级，不拆中心预算**
17. **跨库联动**：finance.db独立，审批通过后跨库写budget.db的cost_record
